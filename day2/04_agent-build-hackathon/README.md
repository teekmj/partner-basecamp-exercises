# Helios RFP Agent — Multi-Agent Response Platform

A production-style AI platform that automates RFP (Request for Proposal) response generation for cybersecurity vendors. What started as a hackathon notebook has grown into a **multi-specialist agent system** with a live web UI, scenario library, eval harness, and HTML/JSON export.

## Quick start — run the platform

```bash
cd ui
pip install anthropic flask flask-cors claude-agent-sdk
export ANTHROPIC_API_KEY=sk-ant-...
python3 app.py
# open http://localhost:8080
```

The UI gives you seven modes:

| Mode | What it does |
|---|---|
| **Run** | Paste a questionnaire, watch the 5-stage pipeline light up live, get answers + reviewer scorecard + downloadable HTML. |
| **Scenarios** | 50 seeded RFP scenarios across 12 fictional clients & 6 verticals. Save, clone, replay any of them. |
| **Search** | Cross-corpus search over the knowledge base, saved scenarios, and previously drafted answers. |
| **Evals** | Run any of four eval suites (`smoke`, `factual`, `edge`, `full`) and inspect archived runs. |
| **Architecture** | In-app architecture explainer for live demos. |
| **Presentation** | Demo mode — large type, slow reveal, talking points generated by a sales-presenter sub-agent. |
| **Dev** | View live `SYSTEM_PROMPT`, recent request log, clear log. |

## Architecture (platform)

```
Browser SPA  ──POST /api/run──▶  Flask  ──▶  Pipeline  ──▶  Anthropic Messages API
   ▲                              │
   └──── Server-Sent Events ──────┘   (stage_start, tool_call, tool_result,
                                       answer_complete, stage_done, …)
```

Five-stage pipeline (`ui/agent_core.py`):

```
PARSE ──▶ RETRIEVE ──▶ DRAFT (multi-specialist) ──▶ REVIEW ──▶ EXPORT
 no LLM     no LLM       LLM + tools                  LLM       no LLM
```

### Multi-specialist routing

Stage 3 routes each question to one of four `claude-agent-sdk` `AgentDefinition` specialists based on category:

| Specialist | Owns | Style |
|---|---|---|
| **Architect** | Detection, latency, encryption, integrations | Quantified, infrastructure-focused |
| **Pricing Lead** | Tiers, discounts, contract terms | Structured tier breakdowns, exact dollars |
| **Compliance** | SOC 2, ISO 27001, FedRAMP, GDPR, audit dates | Enumerates certs with dates & auditors |
| **Business SME** | Customers, references, vertical metrics | NPS, references, vertical context |

Each specialist has its own system prompt, tool list (`search_kb`), and `maxTurns` cap. Ambiguous questions fall back to a heuristic router. See `ui/specialists.py`.

### Parallel execution

The default pipeline drafts the four category specialists in parallel via `ThreadPoolExecutor` (`run_pipeline_parallel`), giving roughly 4× speedup over sequential. The UI exposes this as a checkbox.

### HTML export

`ui/exporter.py` renders any completed run into a self-contained, print-ready HTML document with cover page, executive summary, color-coded answers, reviewer scorecard, and presenter notes. Inline CSS, no external assets, A4-paginated.

## Knowledge base

5 curated entries (`KNOWLEDGE_BASE` in `agent_core.py`):

- Threat detection — latencies, EPS, EPP/EDR vs. SIEM add-on split
- Compliance register — SOC 2, ISO 27001, FedRAMP, PCI DSS, HIPAA, StateRAMP with audit dates
- Pricing — 500/1k/5k tiers, mid-tier interpolation rule, term discounts
- Financial-services customers — 47 customers, 3 references, NPS
- EU data residency — Frankfurt + Dublin, AES-256-GCM, TLS 1.3, GDPR DPA

Retrieval is keyword overlap with a `+5` category bonus — fast, deterministic, debuggable. Embeddings are listed in the production path, not built.

## Eval suites

| Suite | What it covers | Speed |
|---|---|---|
| `smoke` | 1 question, structural shape only | seconds |
| `factual` | 5 sample questions, expected data points (latency, audit dates, prices) | ~minute |
| `edge` | 3 deliberately problematic questions (ambiguous, out-of-scope, typos) | ~minute |
| `full` | 5 sample questions + reviewer consistency check | ~2 minutes |

Each run produces a list of assertions `{test, passed, detail}`. Archived runs are written to `ui/data/evals_archive/` (committed) and surfaced in the Evals tab.

### Stability harness (notebook track)

`specs/001-rfp-agent/stability_test.py` runs 100 trials of the same RFP and aggregates latency, token usage, and reviewer outcomes. Measured baseline: **66s median**, **~$0.50/RFP**, reviewer `consistency_score = high` on 100/100 runs.

### Test coverage (platform)

19 test files under `ui/tests/` covering parse, retrieve, draft, tool-loop, JSON-extraction edge cases, reviewer/exporter, specialists, storage, search, scenarios, eval archive, present mode, and HTTP API (v1/v2/v3). Mocked Anthropic client in `conftest.py` — no API key needed for unit tests.

## What this optimizes for

- **Live feedback over throughput** — SSE stage events make the agent feel responsive even when an LLM call takes 20s.
- **Decoupled stages** — each is independently testable, replaceable, observable.
- **Robustness over cleverness** — JSON extractor handles fenced/embedded/raw output; `max_turns` caps runaway loops; every failure mode has a fallback shape.
- **No build step** — vanilla JS, plain HTML, single `pip install`. `python3 app.py` and you're up.

## What's deliberately out of scope

- Persistence beyond the local JSON store (no DB)
- Authentication (single-user assumption)
- Streaming of LLM tokens themselves (we stream stage events, not partial text)
- KB editing UI (KB is hardcoded in `agent_core.py`)
- Real semantic search (keyword + category; embeddings deferred)
